# Code of Conduct

Ariadne is a scientific / engineering project. We expect contributors to behave like
adult colleagues:

- **Be direct about technical issues.** "This is wrong because X" is welcome. "You are
  wrong because X" is not.
- **Be honest about uncertainty.** "I'm not sure but I think Y" is welcome. "Trust me"
  is not.
- **Be open to being wrong.** When evidence shows you're wrong, update your position.
  When you can't find evidence either way, say so.
- **Be useful.** Bug reports beat complaints; PRs beat wishlists; reproducible
  examples beat anecdotes.
- **Be respectful of the honesty firewall.** Don't represent unverified numbers as
  verified, don't hide limitations, don't paper over null results. Honest negatives
  ARE results.

## Things that won't be tolerated

- Personal attacks, harassment, or discrimination of any kind.
- Misrepresenting Ariadne's capabilities ("Ariadne can replace GMAT" — it can't, see
  the honest-scope sections).
- Bad-faith engagement (rules-lawyering, deliberately misreading clear positions,
  sealioning).
- Sustained off-topic argument in technical issues / PRs.

## What happens if something goes wrong

Open an issue or email the maintainers. We will:

1. Read your report.
2. If the issue is technical: investigate, respond, fix.
3. If the issue is interpersonal: talk to the parties involved. We can ask people to
   step away from a thread, ask for an apology, or in serious cases ban from the
   project. We will not make this decision in public.

## Scope

This code applies to all Ariadne project spaces — issues, PRs, discussions,
correspondence about the project.

## Origin

A trimmed adaptation of the spirit (not the letter) of the Contributor Covenant. The
core rule: be a useful adult colleague.
